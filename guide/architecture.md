# 架构结构

## 生命周期

`bamgoo.Go()` 当前执行顺序：

1. `Load`：通过 config 模块加载配置。
2. `Setup`：模块做默认值和预处理。
3. `Open`：模块建立连接。
4. `Start`：模块开始对外服务。
5. `Wait`：等待退出信号。
6. `Stop`：逆序停止模块。
7. `Close`：逆序关闭资源。

## 调用路径

服务调用优先本地方法，找不到时走 bus 远程调用。

- 本地调用：`core.invokeLocal`
- 远程调用：`hook.Request(...)`

## 文档映射

建议文档也按运行时视角组织：

- 核心：解释生命周期与注册机制。
- 模块：说明模块如何进入生命周期。
- 驱动：说明 `Connect/Open/Start` 等实现差异。
